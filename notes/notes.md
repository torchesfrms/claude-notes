## 📝 监听状态（LISTEN）是什么意思？

**时间**: 2026/02/01 19:13:16

**标签**: `网络` `TCP` `端口` `服务器` `基础知识`

### 💡 回答

**监听 = 等待客户端连接**

就像电话接线员守在电话旁等待来电，或商店开门营业等待顾客进来。

**监听状态表示：**
1. ✅ 程序已经启动
2. ✅ 程序正在某个端口上等待
3. ✅ 随时准备接受连接请求
4. ⏰ 持续等待，不会主动关闭

**完整流程：**
```
服务器启动 → 绑定端口 → 进入 LISTEN 状态 → 等待连接
浏览器访问 → 发送连接请求 → 服务器接受 → ESTABLISHED 状态 → 传输数据
```

**常见的 TCP 连接状态：**
- **LISTEN** - 等待连接（服务器）
- **ESTABLISHED** - 已连接，正在通信
- **TIME_WAIT** - 连接关闭中
- **CLOSE_WAIT** - 等待关闭

**生活比喻：**
披萨店开门（启动程序）→ 打开电话线（绑定端口）→ 等待顾客打电话（LISTEN）→ 接到订单（接受连接）→ 制作披萨（处理请求）

**实际例子：**
```
node 18776 ... TCP localhost:3001 (LISTEN)
```
表示 Node.js 服务器在 3001 端口监听，等待浏览器访问 http://localhost:3001/

**关键要点：**
- LISTEN ≠ 正在传输数据（LISTEN = 等待，ESTABLISHED = 通信）
- 只有服务器才 LISTEN（客户端主动发起连接）
- 一个端口只能有一个程序 LISTEN（否则端口冲突）

**查看监听状态命令：**
- `lsof -i | grep LISTEN` - 查看所有监听端口
- `lsof -i :3001 | grep LISTEN` - 查看特定端口
- `netstat -an | grep LISTEN` - 另一种查看方式

## 📝 目录的格式一般是什么样的？

**时间**: 2026/02/01 17:28:27

**标签**: `文件系统` `路径` `Linux` `Windows` `基础知识`

### 💡 回答

**不同操作系统的格式：**

**Unix/Linux/macOS：**
```
/Users/moer/claude-notes/notes/notes.md
```
- 使用 `/` 作为分隔符
- 以 `/` 开头表示从根目录开始
- 区分大小写（`Notes` ≠ `notes`）

**Windows：**
```
C:\Users\moer\Documents\notes.txt
```
- 使用 `\` 作为分隔符
- 以盘符开头（C:, D:, E: 等）
- 不区分大小写（`Notes` = `notes`）

**绝对路径 vs 相对路径：**

**绝对路径**（从根目录开始）：
- macOS/Linux: `/Users/moer/claude-notes/notes.md`
- Windows: `C:\Users\moer\Documents\file.txt`
- 特点：完整路径，无论从哪里执行都指向同一位置

**相对路径**（从当前目录开始）：
假设当前在 `/Users/moer/claude-notes`：
- `notes/notes.md` - 当前目录下的 notes 文件夹
- `./notes/notes.md` - 同上（明确表示当前目录）
- `../Desktop/file.txt` - 上级目录的 Desktop 文件夹
- `../../Documents/` - 上两级目录的 Documents

**特殊目录符号：**
- `/` - 根目录（Unix）或分隔符
- `~` - 当前用户主目录（`~/Documents` = `/Users/moer/Documents`）
- `.` - 当前目录
- `..` - 上级目录（父目录）
- `-` - 上一个工作目录

**常见错误：**
- ❌ 混淆斜杠：`/Users\moer\notes`
- ✅ 正确：`/Users/moer/notes`
- ❌ 路径有空格忘记引号：`cd /Users/moer/My Documents`
- ✅ 正确：`cd "/Users/moer/My Documents"`

**验证路径的技巧：**
- `ls /path/to/check` - 检查路径是否存在
- `realpath notes/notes.md` - 显示完整路径
- `file /path` - 检查是文件还是目录

## 📝 pwd 命令详解

**时间**: 2026/02/01 17:21:47

**标签**: `命令行` `pwd` `Linux` `路径` `深度学习`

### 💡 回答

**pwd = Print Working Directory（打印工作目录）**
- Print - 打印/显示
- Working - 工作/当前
- Directory - 目录/文件夹

**基本用法：**
```bash
pwd
# 输出：/Users/moer/claude-notes
```

**常用选项：**
- `pwd -L` （默认）- 显示逻辑路径（包含符号链接）
- `pwd -P` - 显示物理路径（解析符号链接）

**路径解析：**
```
/Users/moer/claude-notes
│    │    │         │
│    │    │         └─ 当前目录
│    │    └─ 你的用户文件夹
│    └─ 所有用户的根目录
└─ 系统根目录
```

**实际应用场景：**

1. **确认位置**
   ```bash
   pwd  # 确认我在正确的项目目录
   ```

2. **在脚本中使用**
   ```bash
   CURRENT_DIR=$(pwd)
   echo "当前在：$CURRENT_DIR"
   ```

3. **复制当前路径**
   ```bash
   pwd | pbcopy  # macOS 复制到剪贴板
   ```

**相关命令对比：**
- `pwd` - 显示当前目录
- `ls` - 列出当前目录内容
- `cd` - 切换目录
- `dirname` - 获取路径的目录部分

**快捷技巧：**
- `cd -` - 快速回到刚才的目录
- `OLD_DIR=$(pwd)` - 保存当前目录到变量
- `open .` - 在 Finder 中打开当前目录（macOS）

## 📝 ./start.sh 是什么意思？

**时间**: 2026/02/01 17:10:47

**标签**: `Shell` `命令行` `脚本` `Linux基础`

### 💡 回答

**`./start.sh` = "运行当前目录下的 start.sh 脚本"**

**分解说明**：
- `./` = 当前目录（`.` 表示当前文件夹，`/` 是分隔符）
- `start.sh` = 脚本文件名（`.sh` 表示 Shell 脚本）

**为什么要加 `./`？**
- ❌ 直接输入 `start.sh` → 系统提示"找不到命令"
- ✅ 输入 `./start.sh` → 明确告诉系统"文件在当前目录"

**类比理解**：
就像给地址，"start.sh" 像说"王府井"（位置不明确），而 "./start.sh" 像说"当前文件夹里的 start.sh"（位置明确）。

**start.sh 做什么**：
这个脚本自动完成多个操作：
1. 检查是否安装了依赖
2. 启动后端服务器（3001端口）
3. 启动前端界面（3000端口）
4. 显示访问地址

**相当于把多个命令打包成一键启动！**

**其他相关命令**：
- `cat start.sh` - 查看脚本内容
- `chmod +x start.sh` - 给脚本执行权限
- `Ctrl + C` - 停止运行的脚本

## 📝 为什么启动笔记时端口从 3000 变成了 3001？

**时间**: 2026/02/01 17:05:46

**标签**: `前端开发` `端口` `Vite` `问题排查`

### 💡 回答

**原因：端口冲突自动切换**

**设计初衷**：
- 后端服务器：3001 端口
- 前端界面：3000 端口

**实际情况**：
启动时 3000 端口已被占用，Vite 显示：`Port 3000 is in use, trying another one...`

**发生了什么**：
1. 后端先启动占用了 3001
2. 前端想用 3000，但被其他程序占用
3. Vite 智能地切换到下一个可用端口 3001
4. 结果：前端成功运行在 3001 端口

**解决方案**：
- 方案 1：接受现状，访问 http://localhost:3001/ 即可正常使用
- 方案 2：停止占用 3000 端口的程序，重新启动服务

## 📝 什么是期权？

**时间**: 2026/02/01 17:00:00

**标签**: `金融` `投资` `期权` `基础知识`

### 💡 回答

**期权就像"购买的权利"**，而不是义务。

**生活比喻**：
您看中一套房子，现在100万。您付5万元，获得"3个月内以100万购买这套房的权利"。
- 3个月后房价涨到120万 → 您行使权利，按100万买入，赚15万（扣除5万成本）
- 房价跌到80万 → 您放弃购买，只损失5万

**金融中的期权**：
- **看涨期权（Call）**：买入股票的权利，预期股价上涨时购买
- **看跌期权（Put）**：卖出股票的权利，预期股价下跌时购买

**关键要素**：
1. 标的资产 - 股票、商品等
2. 行权价 - 约定的买卖价格
3. 到期日 - 权利有效期
4. 权利金 - 购买期权的费用

**风险提示**：期权买方最多损失权利金，卖方可能面临巨大损失，适合有经验的投资者。

## 📝 常见的 Zsh 配置文件有哪些？它们的区别是什么？

**时间**: 2026/02/01 16:43:58

**标签**: `Zsh` `配置文件` `终端` `Shell`

### 💡 回答

Zsh 有 4 个主要配置文件：

1. .zprofile - 登录时执行一次，用于设置环境变量、启动程序
2. .zshrc - 每次打开终端都执行，用于设置别名、函数、提示符
3. .zlogin - 登录后执行，类似 .zprofile
4. .zlogout - 退出时执行，用于清理工作

大多数情况下，修改 .zprofile 或 .zshrc 就够了！

.zprofile 适合：环境变量、一次性启动设置
.zshrc 适合：别名、函数、每次都需要的配置

## 📝 如何清除占用端口的进程？

**时间**: 2026/02/02 02:23:41

**标签**: `端口` `进程管理` `命令行` `lsof` `kill`

### 💡 回答

**三步清除端口进程：**

**1. 查找占用端口的进程**
```bash
lsof -i :3000
```
输出示例：
```
COMMAND   PID  USER   FD   TYPE     DEVICE SIZE/OFF NODE NAME
node    34771  moer   19u  IPv6 ...      0t0  TCP localhost:hbci (LISTEN)
```
其中 `34771` 就是进程 ID (PID)

**2. 终止进程**
```bash
kill 34771              # 正常终止（SIGTERM）
kill -9 34771           # 强制终止（SIGKILL）
```

**3. 验证端口已释放**
```bash
lsof -i :3000           # 无输出表示端口已释放
```

**快捷一键命令：**
```bash
lsof -ti :3000 | xargs kill        # 自动查找并终止
lsof -ti :3000 | xargs kill -9     # 强制终止
```

**常用命令汇总：**
- `lsof -i :端口号` - 查找占用特定端口的进程
- `lsof -i | grep LISTEN` - 查看所有监听端口
- `kill PID` - 正常终止进程
- `kill -9 PID` - 强制终止进程
- `killall 进程名` - 按名称终止所有匹配的进程

**kill 信号说明：**
- `kill PID` (SIGTERM) - 礼貌地请求程序退出，允许清理资源
- `kill -9 PID` (SIGKILL) - 强制立即终止，无法被忽略
- `kill -15 PID` (SIGTERM) - 等同于 `kill PID`

**实际应用场景：**
- 端口冲突时清除占用进程
- 开发时重启卡住的服务器
- 释放被意外占用的端口
